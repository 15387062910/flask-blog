做一个网站的大致步骤:
可以下面的步骤以模仿制作一个博客网站为例:
模仿网站链接：https://www.jianshu.com/u/yZq3ZV


1. 拆分有哪些页面 -> 前端工程师来制作相应页面 -> 主要使用HTML CSS JavaScript 当然复杂的项目会用到某些js框架: vue或angular或react
    所有页面都有一个公共的nav header(头部导航条)
    1.1 博客首页 -> 浏览所有的博客 + 发博客
        博客标题 -> 可以点击(点进去就是具体的博文)
    
    1.2 具体博客页面 -> 一篇具体的博文
        博客的标题
        博客的作者(点进去是个人主页)
        博客的发布时间
        博客的内容
        博客的评论(评论最上面是发布评论)
    
    1.3 发布博客页面
    	一个表单 在这填标题和内容 发布博客

    1.4 个人主页
    	显示一些个人的基本信息以及个人进行一些操作

    1.5 登陆页面和注册页面
    	很常见的页面 没什么好说的

    注: 刚刚开始开发我只做了简化版的博客首页和具体博客页面，其他页面todo


2. 建立模型 -> 组织那些数据，把数据的操作实现 -> 采用面向对象的思维抽象出数据和方法
# 注意下面的是简化版的 后期可以将整个模型慢慢优化
Blog(博文):
Blog数据
    1. id
    2. author 
    3. content
    4. create_time
Blog方法
    1. new: new出来一个blog
    2. get_blog: 根据blog id拿到一个blog

BlogComment(博客评论):
BlogComment数据
    1. id
    2. author
    3. comment
    4. create_time
    5. blog_id 也应该被索引 -> 外键
BlogComment方法
    1. new
    2. blog_id find


## 注意:
	网站页面的逻辑应该和数据相互分离，也就是说数据要是独立与页面之外的，不能和页面紧紧捆绑在一起


3. 逻辑 -> 产品经理写文档 写完甩给程序员开发
    3.1 blog首页点创建blog 调到blog/new这个页面，去创建blog
    3.2 创建完毕，回到首页
    3.3 点击某一个blog，进入blog的详细页
    3.4 在详情页，提交一条评论，跳转回详情页，显示评论


4. 开始实现代码 -> 后端程序员自己写 -> 一般会使用某些语言的web框架来实现
	web框架距离:
	Python: flask、django
	Java: Spring、spring MVC
	PHP: thinkPHP、Laravel、Zend Framework
	不推荐使用其他语言写网站 上面三个足够你玩个几年了


5. 剩下的部分一点点补全、一点点迭代开发


// 下面的程序员可以压根不用管 爱咋样咋样
6. 美化页面 -> 视觉工程师


7. 交互细节 -> 交互工程师


注：以上七步只有在腾讯、百度、阿里这样的大公司才有有这所有的步骤，国内中小型公司为了压缩成本，做的事情很野鸡，没有那么规范
